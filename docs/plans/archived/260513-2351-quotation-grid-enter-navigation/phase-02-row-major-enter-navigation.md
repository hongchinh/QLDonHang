# Phase 02: Row-Major Enter Navigation

## Objective

- Implement `Enter` and `Shift+Enter` movement inside the line-items grid using the explicit row-major field order.

## Preconditions

- Phase 1 is complete.
- All editable cells have stable IDs generated by the same helper.

## Tasks

1. Context: inspect existing `onKeyDown` logic in `LineItemsGrid`.
2. Implement: add helper `parseLineCellId(id: string): { field: LineFocusField; rowIndex: number } | null`.
3. Implement: add helper `focusLineCell(field: LineFocusField, rowIndex: number): void`.
4. Implement: add helper `focusLineCellAfterRender(field: LineFocusField, rowIndex: number): void` using `setTimeout(..., 0)`.
5. Implement: add `moveLineFocus(current, direction)` logic:
   - forward inside same row if another field exists.
   - forward from `unit-cost` to next row `product-code`.
   - forward from last row `unit-cost` appends `createEmptyLine(fields.length)` and focuses new row `product-code`.
   - backward inside same row if previous field exists.
   - backward from `product-code` to previous row `unit-cost`.
   - backward from first row `product-code` does nothing.
6. Implement: update the wrapper keydown handler to process:
   - `Enter` without `Ctrl`, `Alt`, or `Meta`.
   - `Shift+Enter`.
   - only when `e.defaultPrevented` is false.
   - only when target parses as a known grid cell.
7. Implement: preserve existing `Insert` and `Ctrl+Delete` handling in the same handler.
8. Verify: run typecheck after implementation.

## Verification

- Commands:
  - `cd frontend && npm run typecheck`
- Expected results:
  - TypeScript passes.
  - Existing Insert/Ctrl+Delete shortcut code remains reachable.

## Exit Criteria

- `Enter` advances through the row-major field order.
- `Shift+Enter` reverses through the same order.
- End-of-row and start-of-row transitions work by computed row index.
- End-of-grid `Enter` appends exactly one new row and focuses its product-code cell.
- Autocomplete-owned `Enter` is not overridden when `e.defaultPrevented` is true.
