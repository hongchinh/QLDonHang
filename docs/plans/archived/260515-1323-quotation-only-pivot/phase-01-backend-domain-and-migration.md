# Phase 01 — Backend domain + EF migration

**Status:** [x] complete

## Objective

Cập nhật domain layer: xóa giá trị enum `ConvertedToOrder`, xóa các enum không dùng (Order/Payment/Document), thêm 3 trường timestamp/audit lên `Quotation`. Cập nhật `Permissions` constants. Tạo EF migration backfill an toàn cho dữ liệu hiện có.

## Tasks

1. Trong `Enums.cs`:
   - Trong `QuotationStatus`, xóa dòng `ConvertedToOrder = 4,` (giữ Draft=1, Sent=2, Confirmed=3, Cancelled=9).
   - Xóa toàn bộ enum `OrderStatus`, `PaymentStatus`, `PaymentMethod`, `DocumentType`.
2. Trong `Quotation.cs`, thêm 3 property sau dòng `Status` hiện tại:
   ```csharp
   public DateTime? ConfirmedAt { get; set; }
   public Guid? ConfirmedByUserId { get; set; }
   public DateTime? CancelledAt { get; set; }
   ```
3. Trong `Permissions.cs`:
   - Xóa hằng `public const string ConvertToOrder = "quotations.convert";` trong nested class `Quotations`.
   - Thêm `public const string CancelConfirmed = "quotations.cancel_confirmed";` trong nested class `Quotations`.
   - Xóa toàn bộ nested class `Orders` (View/Create/Update/Delete/Print/Deliver/Pay/ViewCost).
4. Trong `SalesConfiguration.cs`, trong `QuotationConfiguration.Configure`, thêm trước `b.HasOne(x => x.Customer)`:
   ```csharp
   b.Property(x => x.ConfirmedAt).HasColumnName("confirmed_at");
   b.Property(x => x.ConfirmedByUserId).HasColumnName("confirmed_by_user_id");
   b.Property(x => x.CancelledAt).HasColumnName("cancelled_at");
   b.HasIndex(x => x.ConfirmedAt).HasDatabaseName("ix_quotations_confirmed_at");
   ```
   (Index để filter báo cáo doanh thu nhanh.)
5. Generate EF migration:
   ```
   dotnet ef migrations add QuotationOnlyPivot \
     --project backend/src/OrderMgmt.Infrastructure \
     --startup-project backend/src/OrderMgmt.WebApi
   ```
6. Mở file migration vừa tạo, **append vào cuối `Up()`** (sau khi 3 cột đã được EF auto-generate):
   ```csharp
   // Backfill: Quotation đã ở Confirmed (3) trước migration này thì dùng updated_at làm proxy ConfirmedAt.
   migrationBuilder.Sql(@"
       UPDATE quotations
       SET confirmed_at = updated_at
       WHERE status = 3 AND confirmed_at IS NULL;
   ");

   // Migrate giá trị cũ ConvertedToOrder (4) sang Confirmed (3) + backfill ConfirmedAt nếu trống.
   migrationBuilder.Sql(@"
       UPDATE quotations
       SET status = 3, confirmed_at = COALESCE(confirmed_at, updated_at)
       WHERE status = 4;
   ");

   // Bỏ permission ConvertToOrder khỏi role_permissions để tránh rò rỉ quyền không còn ý nghĩa.
   migrationBuilder.Sql(@"
       DELETE FROM role_permissions
       WHERE permission_id IN (SELECT id FROM permissions WHERE code = 'quotations.convert');
   ");

   // Bỏ toàn bộ permission Orders.* khỏi role_permissions (permission row giữ lại làm orphan; xóa trực tiếp tránh foreign-key issues nếu seeder chạy lại).
   migrationBuilder.Sql(@"
       DELETE FROM role_permissions
       WHERE permission_id IN (
           SELECT id FROM permissions
           WHERE code LIKE 'orders.%' OR code = 'quotations.convert'
       );
   ");
   ```
7. Trong `Down()` của cùng file, thêm trước phần auto-generated drop columns:
   ```csharp
   // Không restore semantic ConvertedToOrder (data đã merge vào Confirmed). Down chỉ drop column.
   ```
8. Verify migration script trước khi apply:
   ```
   dotnet ef migrations script --project backend/src/OrderMgmt.Infrastructure \
     --startup-project backend/src/OrderMgmt.WebApi --idempotent --no-build \
     --output backend/src/OrderMgmt.Infrastructure/Persistence/Migrations/preview.sql
   ```
   Mở `preview.sql`, đảm bảo:
   - Có `ALTER TABLE quotations ADD COLUMN confirmed_at`, `confirmed_by_user_id`, `cancelled_at`.
   - Có 3 block `UPDATE`/`DELETE` ở trên.
   - Sau khi review, xóa file `preview.sql` (không commit).
